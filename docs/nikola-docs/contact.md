## <a href="contact.html#toc-entry-1" class="toc-backref"
role="doc-backlink">Help and Support</a><a href="#help-and-support" class="headerlink"
title="Permalink to this heading">¶</a>

1.  Create a <a href="https://github.com/getnikola/nikola/discussions"
    class="reference external">Discussion on GitHub</a>.

2.  There’s an IRC channel: #nikola on Libera.Chat. You can find many
    developers and volunteers (users) there which are there to provide
    help, if they are around and read your message. The channels are
    <a href="https://irclogs.getnikola.com/"
    class="reference external">publicly logged</a>.

3.  If you are experiencing a bug, you should report it as an Issue on
    GitHub (see below).

### <a href="contact.html#toc-entry-2" class="toc-backref"
role="doc-backlink">Plugins and Themes</a><a href="#plugins-and-themes" class="headerlink"
title="Permalink to this heading">¶</a>

Plugins and Themes from the Indexes are not officially supported.

1.  Create a <a href="https://github.com/getnikola/nikola/discussions"
    class="reference external">Discussion on GitHub</a>, which may or
    may not work out.

2.  You can also try the IRC channel if the theme/plugin is very
    popular.

3.  Report an issue at the
    <a href="https://github.com/getnikola/plugins/issues"
    class="reference external">plugins issue tracker</a> or at the
    <a href="https://github.com/getnikola/nikola-themes/issues"
    class="reference external">themes issue tracker</a>.

4.  Contact the author of the plugin/theme. The name of the author of a
    plugin is available in the `.plugin` and `.py` files. Tracking down
    theme authors can be much harder due to how our architecture works
    (but certainly is doable).

## <a href="contact.html#toc-entry-3" class="toc-backref"
role="doc-backlink">Report a Bug, Request a Feature</a><a href="#report-a-bug-request-a-feature" class="headerlink"
title="Permalink to this heading">¶</a>

Bug reports should be sent to the
<a href="https://github.com/getnikola/nikola/issues"
class="reference external">Issue tracker</a> at GitHub.

Feature requests should be sent to the
<a href="https://github.com/getnikola/nikola/issues"
class="reference external">Issue tracker</a> or in a
<a href="https://github.com/getnikola/nikola/discussions"
class="reference external">Discussion on GitHub</a>, especially if they
require further discussion.
